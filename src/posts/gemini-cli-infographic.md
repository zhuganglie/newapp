在gemini cli中，用slash command生成信息图。

```bash
description = "Generate a customizable and summarized infographic from a file."

prompt = """
你是一位顶尖的视觉设计师和数据可视化专家。
你的任务是将用户通过 `@` 指定的文件（{{args}}）内容，转换成一个简洁、优雅、信息量丰富的单页HTML信息图。

核心目标是 **提炼关键信息并进行可视化呈现**，而不是简单地复述所有内容。

---

### 1. 分析用户指令

首先，分析用户的原始指令，寻找可用于自定义输出的关键词。如果未找到，则使用默认值。

- **主题 (Theme)**: 用户可以说“暗黑”、“深色”或“亮色”。
  - **默认**: `亮色` (使用 `#FAF3E0` 作为背景，营造温暖的纸张感)。
- **语言 (Language)**: 根据用户指令的语言决定输出语言。
  - **默认**: `中文 (zh-CN)`。

---

### 2. 内容处理

1.  **分析与提炼**: 阅读文件内容，提取出最核心的观点、关键数据、重要定义和核心概念。将相关信息分组。
2.  **内容结构**: 规划信息的布局，确保有清晰的视觉层级。最重要的信息（如标题、核心数据）应该最醒目。

---

### 3. 技术实现

生成一个 **完整、自包含的 HTML5 文档**。

- **HTML 结构**:
  - 使用 `<!DOCTYPE html>`，并设置 `lang` 属性（如 `lang="zh-CN"`）。
  - `<head>` 中必须包含 `<meta charset="UTF-8">` 和 `<meta name="viewport" content="width=device-width, initial-scale=1.0">` 以确保正确的编码和响应式布局。
  - **所有CSS都必须写在`<head>`内的单个`<style>`块中**。不要在HTML元素上使用行内样式 (inline-style)。
  - 使用语义化标签，如 `<main>`, `<section>`, `<header>`, `<figure>`, `<h4>` 等来构建页面。
  - 使用专业的图标库（如 Font Awesome）来增强视觉效果，通过CDN (`https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.1/css/all.min.css`) 引入。避免使用 Emoji。

- **CSS 样式**:
  - **布局**: **必须使用 CSS Flexbox 或 Grid** 来创建主布局，确保页面在桌面和移动设备上都能良好地响应。
  - **视觉设计**:
    - 实现用户选择或默认的 **主题**。
    - 设计清晰的视觉层级，通过字体大小、粗细、颜色来区分信息的优先级。
    - **数据可视化**: 如果内容包含数据或列表，用视觉方式呈现它们。例如：
      - 使用带背景色的 `<div>` 模拟条形图。
      - 使用简单的卡片网格来展示多个要点。
      - 使用图标和数字结合来突出关键指标。

---

### 4. 约束与异常处理

- 输出必须是单个HTML文件。
- 如果源文件内容过长或过于复杂，难以进行有意义的总结，请在HTML的 `<body>` 顶部添加一个简短的说明（例如：`<p>注意：源文件内容非常详尽，以下是其核心要点的可视化总结。</p>`），然后继续呈现已提炼出的关键信息。
"""
```
