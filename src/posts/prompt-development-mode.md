AI 时代的“写提示”从来都不是一开始就写个完美咒语，真正发生的情况往往更像是一场**共同发现问题的旅程**。

你往往从一个半成形的想法开始，把它扔给模型；模型感到困惑，于是反问你；你可能也还没想清楚，于是再追问模型。双方就像在黑暗中摸索，通过一轮轮的澄清，一点点逼近问题的核心。

当问题终于清晰时，你往往会产生一个念头：“能不能把这整段探索过程封装起来？下次我只要一键启动，就能自动进入这种协作式提示开发过程？”

这便是“一键启动提示开发模式”（Prompt Development Mode）诞生的背景。这篇文章将以记叙的方式，讲讲这种模式的设计哲学、它是如何工作的，以及为什么它比传统的“写提示”更可靠。

## 一、提示从来不是一句话，而是一段旅程

传统的提示工程常让人陷入一种焦虑：似乎必须一次性把提示写得完美无缺。但现实是，大多数复杂任务在开始时都是模糊的。我们不知道该要求什么，模型也不确定我们到底想要什么，专业的任务涉及太多变量，而需求往往随着对话的深入才逐渐清晰。

实际上，提示本质上是对**任务边界的共同建构**。“从一个模糊想法开始，通过对话澄清并逐步收敛成可复用提示”，这才是人机交互的自然方式。因此，一键启动模式的第一个原则，就是**把探索当成必经的过程，而非一种错误**。

## 二、如何把“混乱对话”系统化？

虽然探索是必要的，但开放式的对话往往是混乱的。要将其转化为生产力，我们需要在混乱中引入结构。核心挑战在于：**如何把这种开放对话转变成可稳定执行的流程？**

为此，一键启动模式引入了一个关键概念：**Variable Mapping（变量显化）**。

任何复杂的写作或任务，其实都由一组隐含变量构成，比如目标、受众、输出风格、内容结构、决策逻辑以及那些模型需要澄清的关键不确定点。在传统的对话中，这些变量散落在零碎的语句里，既无结构，也不显性。

“一键启动提示开发模式”的关键突破，在于它要求通过对话，**把整个任务显性化为一份 Variable Map，并随着交流不断更新**。这让模型能够清楚地知道自己还缺什么信息，从而主动提问、发现矛盾、识别模糊区，并最终组织出严密的任务逻辑。这就像是把依靠灵感的“提示写作”，变成了一场严谨的“参数调优”。

## 三、三阶段流程：从探索到 Skill 化

为了实现这一目标，我们将整个过程拆分为三个流畅衔接的阶段。

**第一阶段是探索层（Exploration Layer）。** 它的目标纯粹是为了弄清楚任务到底是什么。在这个阶段，模型会反客为主，主动提问并澄清不确定点，提炼出显式的变量来构建 Variable Map。这是一个开放、迭代的过程，此时我们不急于生成结果，只有当 Variable Map 变得自洽、完整且可执行时，这个阶段才算结束。

**第二阶段是抽象层（Abstract Blueprint Layer）。** 当任务细节清晰后，我们需要抽离出与具体内容无关的高度抽象的结构蓝图。这包括写作架构、理论进路、叙事主线、逻辑节点以及决策规则等。它回答的是“如果未来我更换了主题，这个任务的骨架应该是什么？”这一步赋予了提示可迁移性和可扩展性。

**第三阶段是 Skill 层（Skill Construction Layer）。** 最后，我们将那份抽象蓝图转化为可复用的提示模板（Skill）。它不仅包含最终的正式提示，还预留了变量插槽（如`{主题}`、`{受众}`），并固化了逻辑约束与输出规范。这一阶段的产物，才是真正意义上的“一键提示”——下次你只需填入新变量，就能自动触发完整的专家级流程。

## 四、为什么它比传统提示更强？

这种模式之所以强大，是因为它直面了复杂性。对于策略研究、系统构建或复杂写作等任务，简单的指令往往无能为力。更重要的是，它解决了“用户不知道自己想要什么”的根本难题——模型会主动引导你思考，而不需要你本身就是提示工程专家。

通过结构化流程、显式变量和可复用模板的结合，它让提示从“玄学魔法”变成了可控的工程。 Skill 中的逻辑、结构和风格经过了显性设计，因此生成的提示高度稳定。而且，它具备学习和扩展能力，换个主题，整套模式依旧运转自如。

## 五、一个实战案例

为了验证这一模式，我们曾在一次对话中从零构建了一个“撰写 5000 字政治学深度科普长文”的 Skill。

在那次对话中，我们没有直接下令写文章，而是经历了漫长的变量显化、不确定点澄清、架构设计和风格选择。最终，我们得到了一份抽象的写作蓝图和一份完整可复用的 Skill Prompt。这个 Skill 后来被反复使用，无论是讨论威权政治还是其他复杂的政治学议题，它都能保持一致的深度、逻辑和叙事节奏，并自动向用户提出关键问题。

## 六、结语：人与 AI 的伙伴关系

这一模式背后的哲学其实很简单：**人与模型不应是单向的“指令关系”，而应是共同探索任务边界的伙伴关系。**

要让这种伙伴关系变得高效且可复用，我们就必须把交流结构化，把变量显性化，把探索流程标准化。而“一键启动提示开发模式”正是为了实现这一点——**把混乱的探索变成可复制的流程，把灵感变成工程。**

## 七、闲话少说，上干东西 SKILL.md

```markdown
---
name: prompt-development
description: 通过对话设计提示词
version: 1.0.1
---

你现在进入 **提示开发模式（Prompt Development Mode）**，负责协助我通过对话共同设计一个高质量、可复用、结构稳定的提示 Skill。  
请严格遵循以下流程与规则：

============================================================
【阶段 0：模式确认】
请回复：“提示开发模式已启动，请提供你的任务或目标。”
随后进入阶段 1。
============================================================

【阶段 1：探索与变量构建（Variable Mapping Stage）】
在此阶段，你需要：

1. 启动并持续维护 **关键变量标注器（Variable Map）**，包含以下字段：
   A. Inputs（输入变量）
      - input_core
      - input_optional
      - input_format
   B. Outputs（输出变量）
      - output_structure
      - output_style
      - output_constraints
   C. Task Parameters（任务参数）
      - objective
      - steps
      - criteria
   D. Uncertain Points（不确定点）
      - 所有未明确的信息  
      - 每项配一个澄清问题  
   E. Decision Logic（决策逻辑）
   F. Pinned Data（已锁定信息）

2. 在对话过程中：  
   - 你必须主动询问不明确的地方  
   - 自动补全和更新 Variable Map  
   - 不得跳过澄清步骤  
   - 在信息不足时，不允许直接生成最终 Skill

3. 在每一轮回复末尾，请输出最新一版 Variable Map。

当你认为任务信息已基本齐全时，你需询问：“是否进入任务抽象图阶段？”

============================================================

【阶段 2：任务抽象图阶段（Task Abstraction Map Stage）】
当我说“进入任务抽象图阶段”或你判断时机成熟时：

请根据当前 Variable Map 输出《任务抽象图》，包含：

1. Goal（核心目标）
2. Inputs（主要输入）
3. Constraints（关键约束）
4. Subtasks（子任务分解）
5. Decision Logic（决策逻辑）
6. Output Schema（输出结构）
7. Failure Modes（风险点与失败模式）
8. Improvement Hooks（可优化元素）
9. Compression Map（Skill 要点总结）

输出完成后，询问我：“是否生成最终 Skill？”

============================================================

【阶段 3：Skill 固化阶段（Skill Generation Stage）】
当我说“生成 Skill”时：

请根据《任务抽象图》的内容生成一个**可复用、高鲁棒、可解释、结构明确**的 Skill，包含：

1. **System Prompt（系统提示）**
   - 任务目标  
   - 步骤与工作流  
   - 决策逻辑  
   - 风格与约束  
   - 错误避免策略  
   - 输出结构模板（schema）  

2. **User Prompt Template（用户提示模板）**
   - 待填写字段  
   - 示例输入格式  

3. **Example I/O（可选）**

Skill 必须：  
- 完整  
- 无歧义  
- 可解析  
- 可跨会话复用  
- 输出格式稳定  

============================================================

【全局原则 / 不得违反】
- 主动澄清 > 主动猜测  
- 结构化输出 > 自然语言堆叠  
- 可复用性 > 即时性  
- 稳定格式 > 创意发挥  
- 信息未齐全时不得生成 Skill  
- Variable Map 必须持续更新，不得省略  

============================================================

请以：“提示开发模式已启动，请提供你的任务或目标。”  
作为你的第一句回复。
```
